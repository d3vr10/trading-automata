# Database Initialization Guide

The database tables need to be created before using the CLI. This is done via Alembic migrations.

## Quick Start

```bash
# Inside the container
docker exec -it trading-automata bash

# Run migrations to create tables
alembic upgrade head

# Verify tables exist
python -m trading-automata.cli status
python -m trading-automata.cli summary
```

## Detailed Steps

### 1. Start the Bot (Creates PostgreSQL)

```bash
docker-compose -f docker/docker-compose.yml up -d
```

Verify PostgreSQL is running:
```bash
docker-compose -f docker/docker-compose.yml ps
# Should show both 'postgres' and 'trading-automata' containers
```

### 2. SSH into Bot Container

```bash
docker exec -it trading-automata bash
```

### 3. Run Alembic Migrations

```bash
# Inside container
alembic upgrade head
```

**Output should show:**
```
INFO  [alembic.runtime.migration] Context impl PostgresqlImpl.
INFO  [alembic.runtime.migration] Will assume transactional DDL.
INFO  [alembic.migration] Running upgrade  -> 001_initial_schema
```

### 4. Verify Database

```bash
# Check tables exist
psql -h postgres -U postgres -d trading-automata -c "\dt"

# Output should show:
#  schema | trades      | table
#  schema | positions   | table
#  schema | health_checks | table
#  schema | performance_metrics | table
#  schema | trading_events | table
```

### 5. Now Use CLI

```bash
# Inside container
trading-cli status
trading-cli summary
```

If `trading-cli` not found:
```bash
pip install -e .
trading-cli status
```

## Troubleshooting

### "relation 'trades' does not exist"

**Cause:** Migrations haven't been run yet

**Solution:**
```bash
docker exec -it trading-automata bash
alembic upgrade head
```

### "psycopg3 connection errors"

**Cause:** PostgreSQL container not fully ready

**Solution:**
```bash
# Wait a few seconds for database to start
sleep 10

# Then run migrations
docker exec -it trading-automata alembic upgrade head
```

### "alembic: command not found"

**Cause:** Alembic not installed

**Solution:**
```bash
docker exec -it trading-automata pip install alembic
alembic upgrade head
```

### Cannot connect to PostgreSQL

**Check:**
```bash
# Verify containers running
docker-compose ps

# Check database logs
docker logs trading-automata-db

# Check if database is ready
docker exec trading-automata-db pg_isready -h localhost
```

## One-Time Setup Command

Run this once after starting the bot:

```bash
docker exec -it trading-automata bash -c "alembic upgrade head && echo 'Database initialized!'"
```

Or if `trading-cli` is installed:

```bash
docker exec -it trading-automata bash -c "pip install -e . && trading-cli status"
```

---

## Verifying Database Creation

Once migrations run, verify tables are created:

```bash
# List tables
docker exec trading-automata-db psql -U postgres -d trading-automata -c "\dt"

# Check trades table schema
docker exec trading-automata-db psql -U postgres -d trading-automata -c "\d trades"

# Count rows
docker exec trading-automata-db psql -U postgres -d trading-automata -c "SELECT COUNT(*) FROM trades;"
```

## Automatic Initialization (Future)

You can add automatic migrations to the Dockerfile:

```dockerfile
# Add this to docker/Dockerfile after installing dependencies
RUN pip install alembic
CMD ["sh", "-c", "alembic upgrade head && python -m trading-automata.main"]
```

This would automatically run migrations when the container starts.

---

## Manual SQL (Alternative)

If Alembic doesn't work, you can run the migration SQL manually:

```bash
docker exec -it trading-automata-db psql -U postgres -d trading-automata -f /app/alembic/versions/001_initial_schema.py
```

Or apply the SQL directly using the migration script in `alembic/versions/001_initial_schema.py`.

## After Database Initialization

You can now:

✅ Run CLI commands: `trading-cli status`, `trading-cli trades`
✅ Query database: `trading-cli query`
✅ View metrics: `trading-cli metrics`
✅ Monitor health: `trading-cli health`

---

## Database Reset (If Needed)

To completely reset the database:

```bash
# Stop containers
docker-compose down -v

# Start fresh
docker-compose up -d

# Reinitialize
docker exec -it trading-automata bash
alembic upgrade head
```

⚠️ **Warning:** This deletes all historical data!
