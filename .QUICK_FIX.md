# Quick Fix Guide - Container Issues

## Issue 1: trading-cli binary not working

**Problem:** `trading-cli: command not found` or module import error

**Quick Fix (Inside Container):**

```bash
# Use python -m instead of trading-cli
python -m src.cli status
python -m src.cli trades
python -m src.cli health
python -m src.cli summary
```

**Or reinstall properly:**

```bash
cd /app
pip install -e .
# Now trading-cli should work
```

## Issue 2: Alembic configuration error

**Problem:** `FAILED: No 'script_location' key found in configuration`

**Solution:** Now fixed! ✅

The `alembic.ini` file has been moved to the project root. Just run:

```bash
cd /app
alembic upgrade head
```

## Complete Setup (Copy & Paste)

```bash
# Inside the container
cd /app

# Run migrations
alembic upgrade head

# Reinstall CLI
pip install -e .

# Test it
trading-cli status
```

Or all in one line:

```bash
docker exec -it trading-bot bash -c "cd /app && alembic upgrade head && pip install -e . && trading-cli status"
```

## Recommended Usage

Inside the container, use these commands:

```bash
# Works immediately (no installation needed)
python -m src.cli status
python -m src.cli trades
python -m src.cli health
python -m src.cli summary
python -m src.cli metrics
python -m src.cli positions
python -m src.cli query

# Or use as binary (after pip install -e .)
trading-cli status
trading-cli trades
# etc...
```

## One-Time Docker Setup

Run this once after starting the container:

```bash
docker exec -it trading-bot bash << 'EOF'
cd /app
alembic upgrade head
pip install -e .
python -m src.cli status
EOF
```

That's it! 🚀
