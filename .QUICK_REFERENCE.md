# Quick Reference - Database & Health Checks

## Start PostgreSQL

```bash
# Docker (recommended)
docker run -d --name trading-automata-db \
  -e POSTGRES_PASSWORD=postgres \
  -e POSTGRES_DB=trading-automata \
  -p 5432:5432 postgres:15

# Or macOS
brew install postgresql@15 && brew services start postgresql@15

# Or Linux
sudo apt-get install postgresql && sudo systemctl start postgresql
```

## Initialize Database

```bash
python -m trading-automata.database.init
```

Expected output:
```
✅ Connected to PostgreSQL
✅ Database schema created successfully
✅ Created tables: ['trades', 'positions', 'performance_metrics', 'trading_events', 'health_checks']
```

## Run Bot

```bash
# Paper trading (safe for testing)
BROKER=alpaca TRADING_ENV=paper python -m trading-automata.main

# Live trading (use with caution)
BROKER=alpaca TRADING_ENV=live python -m trading-automata.main
```

## Monitor Database

```bash
# Connect to database
psql postgresql://postgres:postgres@localhost:5432/trading-automata

# View recent trades
SELECT symbol, strategy, entry_price, pnl_percent FROM trades
ORDER BY entry_timestamp DESC LIMIT 5;

# View health status
SELECT broker, strategy, is_healthy, connection_errors
FROM health_checks ORDER BY checked_at DESC LIMIT 4;

# View performance metrics
SELECT strategy, total_trades, win_rate, profit_factor
FROM performance_metrics ORDER BY metric_date DESC LIMIT 10;

# Exit psql
\q
```

## Configuration

**File: .env**
```env
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/trading-automata
DATABASE_POOL_SIZE=10
DATABASE_MAX_OVERFLOW=20
```

**Precedence:** Environment Variables > .env > config.yml > Defaults

## Key Features

✅ **Database Layer**
- PostgreSQL with psycopg3 (async)
- 5 tables: trades, positions, performance_metrics, trading_events, health_checks

✅ **Trade Recording**
- Automatic insertion on order execution
- Entry price, quantity, order ID saved
- P&L calculated on exit

✅ **Health Monitoring**
- Tracks connection status
- Detects stale data (>5 min without bars)
- Counts errors, implements reconnection
- Saves every 5 minutes
- Logs dashboard every 10 minutes

✅ **Bot Integration**
- Full async support
- Graceful shutdown
- Automatic cleanup

## Troubleshooting

| Problem | Solution |
|---------|----------|
| Connection refused | Start PostgreSQL: `docker start trading-automata-db` |
| Table doesn't exist | Initialize: `python -m trading-automata.database.init` |
| No trades recorded | Check strategies are generating signals |
| DB connection errors | Verify DATABASE_URL in .env |
| Stale data warning | Check broker connection and market hours |

## Useful Commands

```bash
# Database size
psql -c "SELECT pg_size_pretty(pg_database_size('trading-automata'));"

# Backup database
pg_dump -U postgres trading-automata > backup.sql

# Restore database
psql -U postgres trading-automata < backup.sql

# Clear trades (careful!)
psql -U postgres -d trading-automata -c "DELETE FROM trades;"

# View bot logs
tail -f /tmp/trading-automata.log  # if configured

# Check PostgreSQL is running
ps aux | grep postgres
```

## Key Code Snippets

### Record Trade
```python
trade_id = await repo.record_trade_entry(
    symbol="BTC/USD",
    strategy="rsi_atr_trend",
    broker="alpaca",
    entry_price=Decimal("50000"),
    entry_quantity=Decimal("0.1"),
    entry_order_id="order_123"
)
```

### Check Health
```python
check = health_checks.get("alpaca", "rsi_atr_trend")
if check.is_stale():
    logger.warning("Data is stale")
if not check.is_healthy:
    logger.error("Bot is unhealthy")
```

### Get Metrics
```python
metrics = await repo.get_performance_metrics(
    strategy="rsi_atr_trend",
    days=7
)
# Returns: total_trades, winning_trades, win_rate, profit_factor, etc.
```
