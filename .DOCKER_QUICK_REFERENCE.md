# Docker Quick Reference

## Quick Start

```bash
# Configure
cp .env.example .env
nano .env  # Add your API keys

# Start services (bot + PostgreSQL)
docker-compose -f docker/docker-compose.yml up -d

# Follow logs
docker-compose -f docker/docker-compose.yml logs -f trading-bot

# Stop
docker-compose -f docker/docker-compose.yml down
```

## Essential Commands

### Start/Stop

```bash
# Start services
docker-compose -f docker/docker-compose.yml up -d

# Stop services
docker-compose -f docker/docker-compose.yml down

# Restart specific service
docker-compose -f docker/docker-compose.yml restart trading-bot

# View status
docker-compose -f docker/docker-compose.yml ps
```

### Logs

```bash
# Follow bot logs
docker-compose -f docker/docker-compose.yml logs -f trading-bot

# See last 100 lines
docker-compose -f docker/docker-compose.yml logs --tail=100 trading-bot

# See all services
docker-compose -f docker/docker-compose.yml logs
```

### Database

```bash
# Connect to PostgreSQL
docker exec -it trading-bot-db psql -U postgres -d trading_bot

# Run query
docker exec trading-bot-db psql -U postgres -d trading_bot -c "SELECT COUNT(*) FROM trades"

# Backup database
docker exec trading-bot-db pg_dump -U postgres -d trading_bot > backup.sql

# View database size
docker exec trading-bot-db psql -U postgres -d trading_bot -c "SELECT pg_size_pretty(pg_database_size('trading_bot'))"
```

### Commands in Container

```bash
# Initialize database schema
docker exec trading-bot python -m src.database.init

# Run migrations
docker exec trading-bot alembic upgrade head

# View environment variables
docker exec trading-bot env | grep DATABASE
```

### Monitoring

```bash
# CPU/Memory usage
docker stats

# Container details
docker inspect trading-bot

# Service health
docker ps --no-trunc | grep trading-bot
```

## Common Issues

### PostgreSQL won't start

```bash
# Check logs
docker logs trading-bot-db

# Wait longer for startup
docker-compose -f docker/docker-compose.yml restart postgres
sleep 10
docker-compose -f docker/docker-compose.yml ps
```

### Bot can't connect to database

```bash
# Check database is running
docker-compose -f docker/docker-compose.yml ps

# Check connection string in .env
cat .env | grep DATABASE_URL

# Test connection
docker exec trading-bot python -c "import psycopg; psycopg.connect('$DATABASE_URL')"
```

### Permission denied on logs/config

```bash
# Create directories
mkdir -p logs config

# Fix permissions
chmod 755 logs config

# Restart
docker-compose -f docker/docker-compose.yml restart
```

## Cleanup

```bash
# Remove stopped containers
docker container prune

# Remove unused images
docker image prune

# Remove unused volumes (⚠️ deletes data!)
docker volume prune

# Full cleanup (⚠️ removes everything unused)
docker system prune -a
```

## Configuration

```bash
# View current configuration
docker exec trading-bot env | sort

# Update configuration
nano .env
docker-compose -f docker/docker-compose.yml restart trading-bot

# Reload only PostgreSQL config
docker-compose -f docker/docker-compose.yml restart postgres
```

## Services

| Service | Port | Purpose |
|---------|------|---------|
| trading-bot | - | Trading bot app |
| postgres | 5432 | PostgreSQL database |

## Volumes

| Volume | Purpose |
|--------|---------|
| postgres_data | Database persistence |
| ./logs | Bot logs (host mounted) |
| ./config | Configuration files (host mounted) |

## Shortcuts

```bash
# Alias for easier access
alias dc='docker-compose -f docker/docker-compose.yml'

# Then use:
dc up -d        # Start
dc down         # Stop
dc logs -f      # Follow logs
dc ps           # Status
dc exec trading-bot bash  # SSH into container
```
