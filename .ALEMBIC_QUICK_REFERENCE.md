# Alembic Quick Reference

## Setup (One Time)

```bash
pip install -r requirements.txt
python -m src.database.init
```

## Common Commands

### Initialize/Migrate Database
```bash
# Apply all pending migrations
python -m src.database.init

# Or using Alembic directly
alembic upgrade head
```

### Create New Migration
```bash
# When you need to change schema
alembic revision --message "add new_column to trades"

# Edit the file created:
# alembic/versions/NNN_add_new_column_to_trades.py
# Fill in upgrade() and downgrade() functions
```

### Apply Migrations
```bash
# Apply next migration
alembic upgrade +1

# Apply all pending
alembic upgrade head

# Apply specific revision
alembic upgrade 002
```

### Rollback Migrations
```bash
# Rollback one migration
alembic downgrade -1

# Rollback all migrations
alembic downgrade base

# Rollback to specific revision
alembic downgrade 001
```

### Check Status
```bash
# Show current migration version
alembic current

# Show all migrations
alembic history

# Show what would be applied
alembic upgrade --dry-run head
```

## Migration File Template

```python
"""Brief description of what changed.

Revision ID: NNN
Revises: NNN-1
Create Date: YYYY-MM-DD HH:MM:SS.000000

"""
from alembic import op
import sqlalchemy as sa

revision = 'NNN'
down_revision = 'NNN-1'

def upgrade():
    """Changes to apply."""
    op.execute("""
        -- Your SQL here
        ALTER TABLE trades ADD COLUMN my_column VARCHAR(100)
    """)

def downgrade():
    """Undo changes."""
    op.execute("""
        -- Reverse your SQL here
        ALTER TABLE trades DROP COLUMN my_column
    """)
```

## Common SQL Patterns

### Add Column
```python
op.execute("ALTER TABLE trades ADD COLUMN max_loss NUMERIC(20, 8)")
```

### Drop Column
```python
op.execute("ALTER TABLE trades DROP COLUMN max_loss")
```

### Create Index
```python
op.execute("CREATE INDEX idx_name ON table_name(column_name)")
```

### Create Table
```python
op.execute("""
    CREATE TABLE new_table (
        id SERIAL PRIMARY KEY,
        name VARCHAR(255),
        created_at TIMESTAMP DEFAULT NOW()
    )
""")
```

### Drop Table
```python
op.execute("DROP TABLE IF EXISTS old_table")
```

## Workflow

1. **Plan change** - "I need to add max_loss column to trades"
2. **Create migration** - `alembic revision --message "add max_loss column"`
3. **Edit migration** - Write upgrade() and downgrade()
4. **Apply migration** - `alembic upgrade head`
5. **Update code** - Use the new column
6. **Test** - Verify everything works
7. **Commit** - Git commit both migration and code changes

## Important Notes

✓ Always provide both upgrade() and downgrade()
✓ Never edit migrations after they're applied
✓ Use clear, descriptive revision messages
✓ Test rollback: `alembic downgrade -1` then `alembic upgrade +1`

## File Locations

```
alembic/
├── env.py              ← Alembic config
├── alembic.ini         ← Settings
├── script.py.mako      ← Template
└── versions/           ← Your migrations

```

## When Things Go Wrong

### "Migration file not found"
```bash
# Check migration files exist
ls alembic/versions/

# Check current revision
alembic current
```

### "Can't connect to database"
```bash
# Verify DATABASE_URL
echo $DATABASE_URL

# Test connection
psql $DATABASE_URL -c "SELECT 1"
```

### "Need to mark DB as current without running migrations"
```bash
alembic stamp head
```
