# CLI Quick Reference

## Quick Start (Inside Container)

```bash
# SSH into running bot container
docker exec -it trading-automata bash

# Run any command below
python -m trading-automata.cli <command>
```

## Commands at a Glance

| Command | Purpose | Example |
|---------|---------|---------|
| `status` | Show config & risk settings | `python -m trading-automata.cli status` |
| `summary` | 5-line overview | `python -m trading-automata.cli summary` |
| `health` | Connection status | `python -m trading-automata.cli health` |
| `trades` | View trade history | `python -m trading-automata.cli trades --limit 20` |
| `metrics` | Performance stats | `python -m trading-automata.cli metrics` |
| `positions` | Open positions | `python -m trading-automata.cli positions` |
| `query` | Custom SQL queries | `python -m trading-automata.cli query` |
| `schema` | Table structure | `python -m trading-automata.cli schema --table trades` |

## Useful Queries

### Win Rate by Strategy
```sql
SELECT strategy,
       ROUND(100.0 * SUM(CASE WHEN pnl_percent > 0 THEN 1 ELSE 0 END) / COUNT(*), 2) as win_rate
FROM trades
GROUP BY strategy;
```

### P&L by Date
```sql
SELECT DATE(entry_timestamp) as date,
       COUNT(*) as trades,
       ROUND(SUM(pnl_percent), 2) as total_pnl,
       ROUND(AVG(pnl_percent), 2) as avg_pnl
FROM trades
WHERE pnl_percent IS NOT NULL
GROUP BY DATE(entry_timestamp)
ORDER BY date DESC
LIMIT 30;
```

### Symbol Performance
```sql
SELECT symbol,
       COUNT(*) as count,
       ROUND(AVG(pnl_percent), 2) as avg_pnl
FROM trades
GROUP BY symbol
ORDER BY avg_pnl DESC;
```

### Best & Worst Trades
```sql
SELECT * FROM trades WHERE pnl_percent IS NOT NULL ORDER BY pnl_percent DESC LIMIT 5;
SELECT * FROM trades WHERE pnl_percent IS NOT NULL ORDER BY pnl_percent ASC LIMIT 5;
```

## Pro Tips

1. **Combine with tail for live updates**
   ```bash
   watch -n 60 'docker exec trading-automata python -m trading-automata.cli summary'
   ```

2. **Export to CSV**
   ```bash
   docker exec trading-automata python -m trading-automata.cli trades > trades.csv
   ```

3. **Check specific time period**
   ```bash
   # In query mode:
   SELECT * FROM trades
   WHERE entry_timestamp > NOW() - INTERVAL '24 hours'
   ORDER BY entry_timestamp DESC;
   ```

4. **Monitor health every minute**
   ```bash
   while true; do
     docker exec trading-automata python -m trading-automata.cli health
     sleep 60
   done
   ```

5. **Compare two strategies**
   ```bash
   # In query mode:
   SELECT strategy, COUNT(*), AVG(pnl_percent),
          SUM(CASE WHEN pnl_percent > 0 THEN 1 ELSE 0 END)::float / COUNT(*) * 100 as win_rate
   FROM trades
   GROUP BY strategy
   ORDER BY avg_pnl DESC;
   ```

## Environment Variables

```bash
# Override database connection
export DATABASE_URL="postgresql://user:pass@host:5432/db"

# Logging level
export LOG_LEVEL="DEBUG"

# Run CLI with custom env
docker exec -e DATABASE_URL="..." trading-automata python -m trading-automata.cli status
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| "Failed to connect to database" | Check `docker-compose ps` - PostgreSQL should be running |
| No trades showing | Run bot for a few minutes first to execute trades |
| Colors not displaying | Normal in piped output, terminal will show colors |
| Permission denied | Use `docker exec trading-automata` instead of `docker-compose exec` |
