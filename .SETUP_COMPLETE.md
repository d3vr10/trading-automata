# ✅ Automated Setup Complete

Database initialization is now **fully automated** via Alembic in Docker!

## What Changed

**Updated `docker/Dockerfile`:**
```dockerfile
# Before
CMD ["python", "-m", "src.main"]

# After
CMD ["sh", "-c", "alembic upgrade head && python -m src.main"]
```

Now when the container starts, it:
1. ✅ Runs Alembic migrations (`alembic upgrade head`)
2. ✅ Creates all database tables automatically
3. ✅ Starts the trading bot

## How to Use (Super Simple!)

```bash
# 1. Start containers
docker-compose -f docker/docker-compose.yml up -d

# 2. Wait ~5 seconds for database to initialize

# 3. SSH into bot
docker exec -it trading-bot bash

# 4. Use CLI immediately - tables are ready!
python -m src.cli status
python -m src.cli summary
python -m src.cli trades
```

**That's it!** No manual `alembic upgrade head` needed. ✅

```

3. Alembic runs: alembic upgrade head
```

## Troubleshooting

### "relation 'trades' does not exist"

**Cause:** Container didn't finish initialization yet

**Solution:** Wait 5-10 seconds after `docker-compose up -d`

```bash
sleep 10
docker exec trading-bot python -m src.cli status
```

### "alembic: command not found"

**Unlikely** since Alembic is in requirements.txt, but if it happens:

```bash
docker exec trading-bot pip install alembic
```

### Container keeps restarting

**Check logs:**
```bash
docker logs trading-bot
```

If it's a database connection issue, make sure PostgreSQL is running:
```bash
docker-compose ps
```
