# Trading Bot CLI Guide

Command-line interface for querying trading bot data and status without needing Telegram or web dashboard.

## Installation

CLI is included in `requirements.txt` with:
- `click>=8.0.0` - Command framework
- `tabulate>=0.9.0` - Pretty table formatting

## Quick Start

### Run Inside Docker Container

```bash
# SSH into running container
docker exec -it trading-bot bash

# Run CLI commands
python -m src.cli status
python -m src.cli trades --limit 20
python -m src.cli metrics
python -m src.cli health
python -m src.cli summary
```

### Run on Host (requires database connection)

```bash
# Set environment variables for database
export DATABASE_URL="postgresql://postgres:postgres@localhost:5432/trading_bot"

# Run CLI
python -m src.cli status
./trading-cli trades
./trading-cli metrics
```

**Useful Queries:**

```sql
-- Get all trades for a symbol
SELECT * FROM trades WHERE symbol = 'SPY' ORDER BY entry_timestamp DESC;

-- Get win rate by strategy
SELECT strategy,
       COUNT(*) as total,
       SUM(CASE WHEN pnl_percent > 0 THEN 1 ELSE 0 END) as wins
FROM trades
GROUP BY strategy;

-- Get daily P&L
SELECT DATE(entry_timestamp) as date, SUM(pnl_percent) as daily_pnl
FROM trades
GROUP BY DATE(entry_timestamp)
ORDER BY date DESC;

-- Get best performing symbols
SELECT symbol, AVG(pnl_percent) as avg_pnl, COUNT(*) as count
FROM trades
GROUP BY symbol
ORDER BY avg_pnl DESC;

-- Get largest losses
SELECT id, symbol, strategy, pnl_percent, entry_timestamp
FROM trades
WHERE pnl_percent IS NOT NULL
ORDER BY pnl_percent ASC
LIMIT 10;
```

## Usage Patterns

### Daily Monitoring

```bash
# Morning check
python -m src.cli summary
python -m src.cli health
python -m src.cli trades --limit 5

# Check specific symbol
python -m src.cli trades --symbol BTC/USD --limit 20
```

### Performance Analysis

```bash
# Overall metrics
python -m src.cli metrics

# Query trades by date range
python -m src.cli query
# Then run:
# SELECT DATE(entry_timestamp), COUNT(*), AVG(pnl_percent)
# FROM trades
# WHERE entry_timestamp > NOW() - INTERVAL '7 days'
# GROUP BY DATE(entry_timestamp);
```

### Troubleshooting

```bash
# Check health status
python -m src.cli health

# View error logs
python -m src.cli query
# SELECT * FROM trading_events WHERE event_type = 'ERROR' ORDER BY timestamp DESC LIMIT 20;

# See connection errors
python -m src.cli query
# SELECT broker, strategy, connection_errors FROM health_checks;
```

### Exporting Data

```bash
# Get all trades as a formatted table
python -m src.cli trades --limit 1000 > trades_export.txt

# Get specific date range (via query)
python -m src.cli query
# COPY (
#   SELECT * FROM trades
#   WHERE entry_timestamp BETWEEN '2026-01-01' AND '2026-02-01'
# ) TO STDOUT WITH CSV HEADER;
```

## Tips & Tricks

### Get Help

```bash
python -m src.cli --help           # Overall help
python -m src.cli trades --help    # Command-specific help
```

### Pretty Output

All tables use ANSI color codes for better readability:
- 🟢 Green = Healthy/Good metrics
- 🔴 Red = Unhealthy/Bad metrics
- 🟡 Yellow = Warning
- 🔵 Blue = Neutral info

### Performance

For large result sets, the CLI streams data efficiently:
- No memory issues with thousands of trades
- Interactive query mode for exploration
- Schema browsing for understanding structure

## Troubleshooting

### "Failed to connect to database"

Make sure:
1. PostgreSQL is running: `docker-compose ps`
2. DATABASE_URL is correct
3. Bot container is running: `docker ps`

### "No metrics available yet"

The bot needs to execute some trades first. Check:
```bash
python -m src.cli trades
```

### Color codes not showing in output

The CLI auto-detects terminal capabilities. For piping output:
```bash
# Colors won't show in files, that's normal
python -m src.cli summary > summary.txt

# To disable colors in terminal
unset TERM
python -m src.cli summary
```
