# TradingAutomata CLI - Installation & Setup

Make the CLI available as a system command: `trading-cli`

## Installation Methods

### Method 1: Development Install (Recommended)

Install the package in development mode to create the `trading-cli` command:

```bash
# From project root
pip install -e .

# Verify installation
trading-cli --help
```

**What it does:**
- Creates `trading-cli` command available system-wide
- Links to source code (changes immediately take effect)
- No need to reinstall after code changes

### Method 2: Direct Script (No Installation)

Run the executable script directly:

```bash
# From project root
./trading-cli status
./trading-cli trades --limit 20
./trading-cli health
```

**Pros:**
- No installation needed
- Works immediately
- Perfect for Docker containers

**Cons:**
- Must run from project directory
- Requires `./` prefix

### Method 3: Add to PATH

Make script available from anywhere:

```bash
# Create symlink to /usr/local/bin
sudo ln -s /path/to/trading-automata/trading-cli /usr/local/bin/trading-cli

# Now use from anywhere
trading-cli status
trading-cli trades
```

**Requirements:**
- Requires sudo access
- Must use absolute path in symlink

### Method 4: Docker Container

Use inside running container:

```bash
# SSH into container
docker exec -it trading-automata bash

# Method A: Using pip install
pip install -e .
trading-cli status

# Method B: Using script directly
python trading-cli status
python -m trading-automata.cli status
```

## Setup by Environment

### Local Development

```bash
# 1. Clone/navigate to project
cd trading-automata

# 2. Install dependencies
pip install -r requirements.txt

# 3. Install package in development mode
pip install -e .

# 4. Verify it works
trading-cli --help
```

### Docker Container

Add to Dockerfile to make CLI available:

```dockerfile
# Add to docker/Dockerfile after pip install -r requirements.txt
RUN pip install -e .

# Now trading-cli is available in container
CMD ["trading-cli", "status"]  # or any command
```

Or use inside running container:

```bash
docker exec -it trading-automata bash
pip install -e .
trading-cli status
```

### Virtual Environment

```bash
# Create venv
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate

# Install package
pip install -e .

# Use CLI
trading-cli status
```

## Troubleshooting

### "trading-cli: command not found"

**Solution:** Install in development mode:
```bash
pip install -e .
```

### "No module named 'src'"

**Solutions:**
- Run from project root directory
- Or use: `python -m trading-automata.cli`
- Or install with pip: `pip install -e .`

### "Permission denied" when running ./trading-cli

**Solution:** Make it executable:
```bash
chmod +x trading-cli
```

### Database connection errors

**Solution:** Ensure you have the right environment:
```bash
export DATABASE_URL="postgresql://user:pass@host:5432/trading-automata"
trading-cli status
```

### Inside Docker container

```bash
# Install pip dependencies
docker exec trading-automata pip install -e .

# Now use
docker exec trading-automata trading-cli status
```

## Entry Point Configuration

The CLI is configured in `pyproject.toml`:

```toml
[project.scripts]
trading-cli = "trading-automata.cli:cli"
```

This creates a `trading-cli` command that points to the `cli()` function in `trading-automata.cli`.

When you run `pip install -e .`, setuptools creates the command wrapper automatically.

## Uninstall

To remove the CLI:

```bash
# If installed with pip install -e .
pip uninstall trading-automata

# If using symlink
sudo rm /usr/local/bin/trading-cli

# Script still works
./trading-cli status
```
